LIMITATIONS & CONSTRAINTS
----------------------------------------------------------------------
  1. NetworkBehaviour requires NetworkObject component on same or parent GameObject
  2. NetworkVariables only sync from server to clients (owner can write with permissions)
  3. RPCs have naming convention: ServerRpc suffix for client->server, ClientRpc for server->clients
  4. WebGL requires WebSocket transport - UDP not supported in browsers
  5. Spawning must use NetworkManager.Singleton.SpawnManager for proper registration
  6. Object pooling requires INetworkPrefabInstanceHandler implementation
  7. Ownership transfer uses ChangeOwnership() - only server can transfer
  8. Host is both server AND client - check IsServer/IsClient/IsHost appropriately
  9. Hotspot interactions require ownership validation before execution
  10. Player targeting uses NetworkObjectId or ClientId - never direct references across network
  11. NPC spawning uses server-authoritative spawn with client visual sync
  12. Late joiners receive full state sync via NetworkVariable OnValueChanged
  13. Connection approval can filter clients before full connection
